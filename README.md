# API Security Risk Analysis

## Overview:
- This project presents a professional API Security Risk Analysis performed on a publicly available demonstration API. The assessment follows a read-only methodology, focusing on identifying common API security risks without exploiting vulnerabilities or affecting service availability.
- The objective is to demonstrate practical API security assessment skills aligned with modern Application Security (AppSec) and Security Analyst workflows.

## Objectives:

- Perform a structured security assessment of a public REST API.
- Analyze API endpoints using Postman.
- Review authentication and authorization mechanisms.
- Inspect request and response headers.
- Identify potential API security risks.
- Classify findings based on severity.
- Recommend practical remediation strategies.
- Produce a professional security assessment report.

## Scope
- This assessment includes only ethical, non-intrusive testing activities.

### In Scope

- Public API documentation review
- Read-only API requests
- HTTP request and response analysis
- Header inspection
- Authentication review
- Authorization assessment
- Risk identification
- Security recommendations

### Out of Scope

- Exploitation attempts
- Authentication bypass
- Denial-of-Service testing
- Brute-force attacks
- Production system attacks
- Unauthorized access attempts

## Methodology
- The assessment followed the workflow below:
1. Review API documentation.
2. Identify available endpoints.
3. Send read-only API requests.
4. Inspect request headers.
5. Analyze response headers.
6. Review response data.
7. Assess authentication requirements.
8. Evaluate authorization controls.
9. Identify potential security risks.
10. Recommend mitigation strategies.
11. Document findings.

## Security Areas Reviewed

- Excessive Data Exposure
